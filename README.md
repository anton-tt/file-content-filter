# File Content Filter
## Утилита фильтрации содержимого файлов.

Программа принимает от пользователя файлы с текстовым содержимым. Считывает из них данные (в качестве разделителя 
элементов выступает перенос строки), разделяет их на три категории - целые числа, дробные числа (в т.ч. 
в экспоненциальной форме) и строки. После чего сохраняет эти группы данных по умолчанию в соответствующие файлы - 
integers.txt, floats.txt, strings.txt .

Для реализации программы использовался язык программирования Java (версия 11) и автоматизированная система сборки 
Apache Maven (версия 3.9.4) . При первом запуске утилиты необходимо в терминале зайти в папку с программой и запустить 
сборку - командой __mvn package__ . В случае возникновения ошибки повторить сборку - командой **mvn clean package** .

Для запуска утилиты, находясь в папке с программой (*file-content-filter*), необходимо ввести в командную строку - 
команду __java -jar target/util.jar__

Для корректной работы программы в режиме минимальной функциональности необходим хотя бы один файл с данными. В настоящее 
время тестовая версия утилиты умеет обрабатывать файлы с расширением .txt. При запуске по умолчанию считается, что 
файлы, которые необходимо обработать, уже загружены в подпапку src/main/resources. В этом случае, достаточно добавить 
через пробелы после стартовой команды имя/имена этих файлов. Например, имеющийся тестовый файл in1.txt: 
java -jar target/util.jar in1.txt .  
Также при старте можно указать утилите абсолютный путь к файлу, расположенному в другой папке, пример команды:  
java -jar target/util.jar C:\Users\user\Documents\Java\file-content-filter\src\main\resources\in1.txt

При необходимости можно выбрать дополнительные опции для настройки фильтрации файлов (используются символы латиницы,
допустимо введение и до перечисления файлов и после них):

1. **-a** По умолчанию утилита перезаписывает существующий итоговый файл, при этом имеющаяся в нём информация удаляется. 
Данная опция добавляет новые данные к уже имеющимся. Пример команды: java -jar target/util.jar -a in1.txt 

2. **-s** Вывод краткой статистики по работе утилиты. Пользователю сообщается в консоли, сколько элементов было записано 
в каждый итоговый файл, соответствующий одному из типов данных. Пример команды: java -jar target/util.jar -s in1.txt

3. **-f** Вывод полной версии статистики по работе утилиты. Пользователю сообщается в консоли, сколько элементов было 
записано в каждый итоговый файл, также для целых и дробных чисел максимальное и минимальное значения элементов, сумма и 
среднее значение. Для строк выводится размер самой короткой и самой длинной. Пример команды: 
java -jar target/util.jar -f in1.txt

4. **-p** Опция позволяет добавить к именам итоговых файлов префикс. Для корректной работы команды необходимо записать 
после названия опции -р через пробел необходимый префикс с символом _ . Пример команды: 
java -jar target/util.jar in1.txt -p result_  
*Примечание: В имени файла недопустимы следующие символы ? < > " * + : | / \ . В случае выбора данной опции и введения 
некорректного префикса, файлы будут сохранены по умолчанию для данной команды - с префиксом default*_ .

5. **-o** По умолчанию утилита сохраняет итоговые файлы в подпапку src/main/resources. При выборе данной опции можно 
указать программе другую папку для сохранения. Для корректной работы программы команды необходимо добавить после 
названия опции -o через пробел путь к необходимой папке. Пример команды:  
java -jar target/util.jar in1.txt -o src\main\resources - относительный путь.  
java -jar target/util.jar in1.txt -o C:\Users\user\Documents\Java\file-content-filter\src\main\resources - абсолютный 
путь.  
*Примечание: В случае выбора данной опции и введения некорректного пути к папке, файлы будут сохранены по умолчанию для 
данной команды в папке src/main/resources с префиксом* . 

При запуске утилиты допустимо введение нескольких опций после базовой команды java -jar target/util.jar в корректной 
форме. Последовательность опций и файлов неважна, но нельзя разделять команды, например, -o и путь к папке должны быть 
рядом через пробел.  
Пример команды: java -jar target/util.jar in1.txt -o src\main\resources -f -p result_

*Сторонние библиотеки, которые использовались в программе, описаны в файле pom.xml . Работа утилиты проверялась 
в условиях ОС Windows 11.*

*В случае, если ни один файл с данными не будет найден программой, работа утилиты будет закончена.  
В случае, если в файлах не будет найдено никаких данных, работа утилиты будет закончена без формирования итоговых 
файлов.  
В случае, если при активации опции -p не будет задан префикс, файл будет сохранён без префикса.  
В случае, если при активации опции -o не будет задана новая папка, файл будет сохранён в папку src\main\resources.  
В случае, если при фильтрации данных не буден получен один из типов данных, итоговый файл по нему не формируется / 
не перезаписывается.*
